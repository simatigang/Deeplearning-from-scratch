# 📘 '밑바닥부터 시작하는 딥러닝' 학습 기록
> **작성자:** 강동석


## 📝 학습 방향
교재에서 제공하는 예제를 활용해서 딥러닝의 하부구조를 외부 라이브러리를 거의 사용하지 않고 numpy 및 파이썬 내장함수를 이용해서 알고리즘을 구현합니다. 또한, 예제 코드를 실행해보는 것을 넘어서 딥러닝 및 신경망 이론을 제가 이해한대로 정리해봤습니다.

## 🛠 주요 학습 내용

### 퍼셉트론(Chapter 2)
- 퍼셉트론: 다수의 신호를 입력받아 하나의 신호를 출력하는 알고리즘. 각 입력신호에 가중치를 곱하고 그 합이 입계값을 넘으면 1을 출력하게 됨.
- 논리회로 구현: 매개변수 값을 바꾸면 동일한 구조의 퍼셉트론이 다른 논리 게이트로 동작하게 만들 수 있음.
- XOR 문제: 단층 퍼셉트론을 사용하면 AND, NAND, OR을 구현할 수 있지만 XOR 게이트는 만들 수 없음. 단층 퍼셉트론은 직선하나로 영역을 나누는 선형 영역만 분리할 수 있지만, XOR 게이트는 비선형으로 영역을 구분지어야 함. 이를 위해서 AND, NAND, OR을 조합한 다층 퍼셉트론을 이용하면 XOR을 구현할 수 있다. 즉, 이러한 다층 퍼셉트론 아이디어가 다른 복잡한 회로를 만들 수 있게 됨.
- **Insight**: 2장에서는 퍼셉트론이 논리 게이트를 구현하는 과정을 확인했다. 특히 가장 핵심적인 XOR 문제를 해결할 때 디지털 회로설계에서 기본 게이트를 조합해 combinational한 회로를 만드는 과정과 유사하여 흥미로웠다. 딥러닝의 강력함이 비선형성과 다층 구조에서 온다는 근본적인 특성을 이해할 수 있었다.

### 신경망(Chapter 3)
- 활성화 함수: 입력신호의 총합을 출력신호로 변환하는 함수임. 퍼셉트론은 Step function, 신경망은 연속적인 변화를 주는 Sigmoid나 ReLU 함수를 사용함.
- 비선형성: 신경망(다층 퍼셉트론)에서는 비선형 함수를 활성화함수로 사용함. 선형함수를 쓰면 아무리 층을 쌓아도 하나의 선형함수로 표현 가능하기 때문.
- Sigmoid: 출력이 0과 1사이, 부드러운 곡선
- ReLU: h(x) = max(0,x)로 최근 딥러닝에서 가장 많이 쓰고, 계산이 단순해서 연산 효율이 높음. HW 관점으로 봤을 때도 지수함수 연산이 필요한 시그모이드에 비해 단순 비교 연산인 ReLU가 리소스 최적화 및 구동 속도 측면에서 유리할 것임.
- 넘파이 다차원 배열을 이용해서 신경망을 구현할 수 있음. 각 층에서 행렬의 곱 연산을 수행하게 됨.
- Softmax: 클래스 분류 문제에서 출력층에 사용됨. 출력값의 총합이 1이되어 확률로 해석할 수 있게 함. 반면 회귀 문제에서는 출력층에 Identity function을 사용함.
- **Insight**: 3장에서는 퍼셉트론에 활성화 함수를 사용하여 데이터의 연속적 특징을 학습하는 신경망의 기본 구조를 익혔다. 넘파이 행렬 연산을 이용해서 뉴런 연산이 결국 거대한 행렬 곱으로 이루어짐을 확인했다. 영상 처리 측면에서 픽셀 데이터를 효율적으로 처리하기 위해 이런 선형대수적 접근이 필수적인지 이해하게 되었다.

### 신경망 학습(Chapter 4)
- 데이터 주도 학습: 기존 방식에서는 사람이 직접 특징을 찾아 설계하는데, 딥러닝은 데이터로부터 가중치 Parameter를 스스로 찾아냄.
- Loss Function: 신경망 학습에 사용하는 지표. 오차 제곱합, 교차 엔트로피 오차, 등을 사용함.
- Gradient Descent: 손실함수의 값을 최소화 하는 가중치를 찾기 위함. 함수의 미분값을 따라 조금씩 이동하며 최적의 값을 찾는 알고리즘임.
- 수치 미분: 미분값(기울기)를 구하기 위함. 작은 차분(h)를 이용해서 근사적으로 기울기를 계산함. 계산량이 많고 오차가 많은 한계가 있음.
- Mini-batch Training: 수 만개의 데이터를 한번에 계산하기엔 리소스가 부족하므로, 데이터의 일부만 무작위로 뽑아 학습하는 방식임.
- **Insight**: 4장을 통해 신경망이 스스로 학습하기 위해 사용하는 손실함수와 이를 최소화하는 경사하강법의 원리를 파악했다. 수치 미분을 코드로 구현해보면서 수치해석적 접근 방식의 효율성을 체감했다. 또한, 대용량 데이터를 처리하기 위한 미니배치 학습의 필요성을 보며, 메모리 제약이 있는 HW 환경에서의 데이터 처리에 대한 전략도 고민할 수 있었다.

### 오차역전파법(Chapter 5)
-  Forward Propagation(순전파): 데이터를 입력층에서 출력층 방향으로 전달하며 결과값을 계산함.
-  Backward Propagation(역전파): 출력층에서 얻은 Loss를 반대 방향으로 전파하여 각 노드의 기울기를 구함. Chain Rule을 이용했다고 볼 수 있음.
-  Chain Rule: 전체 미분 방정식을 풀지 않고, 국소 미분값만을 알면 전체 기울기를 구할 수 있게 됨.
-  Layer 구현: 활성화 함수의 미분을 구현할 때 ReLU나 Sigmoid를 사용함. Softmax-with-Loss는 출력층에서 소프트맥스와 손실함수를 하나로 묶어 역전파 값이 깔끔하게 나오게 함.
-  **Insight**: 5장에서는 계산 그래프를 사용해서 오차역전파법을 학습했는데, 국소적 미분을 전달하는 과정이 매우 흥미로웠다. 수치미분을 사용할 때와 오차역전파법을 사용할 때의 결과를 비교해보면 오차역전파법을 사용할 때에 속도가 비약적으로 빨라지는 것을 확인할 수 있다. 이를 통해 알고리즘의 최적화가 HW 최적화에 큰 영향을 준다고 다시금 깨달았다.

### 학습 관련 기술들(Chapter 6)
- SGD: 기울기가 가파른 방향을 따라가기 때문에, 특정 함수에 따라 매우 비효율적일 수 있음.
- Momentum: 관성을 도입해서 SGD에서 나타나는 지그재그 현상을 줄임.
- AdaGrad/Adam: 학습률을 조절하여 효율적으로 도달하도록 함.
- 초깃값: Xavier는 sigmoid나 곡선 함수에 특화되어있는 초깃값이고, He 초깃값은 ReLU 함수를 사용하는 데에 특화되어 있다. 초깃값이 0이나 층마다 동일한 값이면 학습이 되지 않음.
- Batch Normalization: 학습이 원활하게 되도록 각 층의 활성화를 적당히 퍼뜨리기 위함. 학습속도 향상, 초기값 의존도 낮아짐, 과적합 억제.
- 하이퍼파라미터 값: 과적합을 막기 위해 시험 데이터는 사용 금지. 검증데이터를 이용하여 최적화.
- **Insight**: 6장에서는 같은 모델이어도 초깃값 설정, 하이퍼파라미터 최적화에 따라 학습 속도와 성능이 달라질 수 있다는 것을 확인했다. 이전 해커톤에서 훈련데이터에만 치중된 모델 때문에 어려움을 겪었는데, 이를 극복할 수 있는 법을 확인할 수 있었다.

### CNN(Chapter 7)
- CNN: 일반적인 Fully connected 방식은 1차원 데이터 형식이기 때문에 데이터의 형상이 무시됨. CNN에서는 이미지의 형상을 유지하면서 패턴 추출 가능.
- Convolution: 입력 데이터와 필터 간 합성곱 연산 수행. Padding과 Stride를 통해 가장자리의 손실을 막음.
- 풀링 계층: 가로 세로 방향의 공간을 줄여서 데이터 압축, 연산량 감소.
- LeNet --> Alexnet
- **Insight**: 7장은 이미지 데이터 처리에 특화된 CNN 구조를 학습했다. 단순 알고리즘 구현을 넘어 영상 데이터를 다루는 데에 필수적인 구조 및 기술이기 때문에 밀도있게 학습하였다.

### 딥러닝(Chapter 8)
- Deeper: 층을 깊게 쌓아서 추상적인 특징을 계층적으로 학습. ResNet에서는 스킵 연결을 통해서 층이 깊어지면 발생하는 문제 해결.
- Faster: 비트 감축, 분산 학습, GPU를 활용해서 딥러닝을 조금 더 빠른 속도로 만듦.
- Application: 이미지 생성, 강화학습(알파고), 분할 등에 활용.
